# TaskDetailPage 布局重设计

**日期**: 2026-03-21  
**状态**: 已批准，待实现

## 问题

当前 `TaskDetailPage` 布局存在以下问题：

1. **左侧空旷**：文件树列（`xl:col-span-5`）仅有 1-2 个文件时大量空白
2. **信息条松散**：7 个字段水平铺开，视觉密度低
3. **时间戳割裂**：单独一行悬浮在信息条下方，与主内容脱节
4. **路径截断**：`max-w-xl` 限制，宽屏下路径后半段被截断

## 目标

- RecoveryPanel 作为主视觉中心，获得更多横向空间
- 元信息与文件树收纳到右侧紧凑侧边栏
- 页面更紧凑，减少无效空白
- 响应式：`xl`（1280px）以下降级为单列

## 方案：紧凑单列主区 + 右侧固定侧边栏

### 整体结构

```

│ ← 返回任务列表                    [JSON] [CSV]  [删除]   │
├─────────────────────────────────────────────────────────┤

```

### 响应式行为

| 窗口宽度 | 布局 |
|---|---|
| `≥ xl (1280px)` | 双列：主区域 `flex-1` + 右侧栏 `w-72` |
| `< xl` | 单列：标题 → 元信息卡 → 错误 → 文件树 → RecoveryPanel（顺序堆叠） |

## 变更范围

仅修改 `src/pages/TaskDetailPage.tsx`，不涉及 `RecoveryPanel.tsx`。

### 具体改动

1. **移除**：独立的 `综合信息条` card（原 `<div className="rounded-lg border bg-card...">`）
2. **移除**：独立的时间戳行（`<div className="flex gap-6 text-sm text-muted-foreground">`）
3. **新增**：右侧侧边栏组件 `<aside>`，包含：
   - 元信息列表（label/value 两列 grid）
   - 错误信息 banner（条件渲染）
   - 归档内容可折叠 section
4. **调整**：主区域改为 `<div className="flex xl:flex-row flex-col gap-6 items-start">`
5. **调整**：左侧主列 `flex-1 min-w-0`，右侧 `w-72 flex-shrink-0`
6. **调整**：路径段移除 `max-w-xl` 限制，改为 `break-all` 允许换行
7. **保留**：双列响应式降级时右侧侧边栏整体移到底部

## 不变更

- `RecoveryPanel.tsx` 内部逻辑和样式
- 路由、状态管理、导出功能
- 所有 i18n key
